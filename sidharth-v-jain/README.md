# AI Startup Validator

## What it does
For the Sprint B project, I decided to implement a site that lets you describe a startup idea in one line, and get back a full validation report, consisting of competitors, a SWOT analysis, a target customer profile, revenue ideas, landing page copy, and a phased roadmap.
The response is generated by an LLM and rendered as a readable, tabbed report.

## Tech Stack
- Backend: FastAPI (Python), rate-limited with slowapi, response validation with Pydantic
- Frontend: Streamlit
- LLM: Groq (`llama-3.3-70b-versatile`)
- Other libraries used: `python-dotenv`, `groq`, `requests`

## How to run locally

**Backend**
```bash
cd backend
python -m venv venv && source venv/bin/activate   
pip install -r requirements.txt
cp .env.example .env        # add GROQ_API_KEY
uvicorn backend:app --reload
```
API runs at `http://localhost:8000` (docs at `/docs`).

**Frontend**
```bash
cd frontend
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
mkdir -p .streamlit
cp .streamlit/secrets.toml.example .streamlit/secrets.toml   # set BACKEND_URL
streamlit run app.py
```

## Environment Variables
Refer to `backend/.env.example` for required keys (`GROQ_API_KEY`, plus optional `BACKEND_API_KEY` and `ANALYST_PERSONA`), and `frontend/.streamlit/secrets.toml.example` for the frontend's `BACKEND_URL` / `BACKEND_API_KEY`.
